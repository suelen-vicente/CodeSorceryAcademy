Building Your First Endpoint with Real Data

In Part 6, we created our first Spring Boot project and a basic endpoint that returned hardcoded data. Now, we’ll connect our FinBank API to a real database and build a CRUD (Create, Read, Update, Delete) endpoint for Customer data.

What Is The Right Order?

When building an API with Spring Boot and Spring Data JPA, there’s a logical flow:

  1. Model your data → Create an Entity (maps to a database table).
  2. Provide access to the data → Create a Repository (interface to the database).
  3. Add business rules → Create a Service (contains the logic).
  4. Expose it via HTTP → Create a Controller (handles requests and responses).

This “inside out” approach keeps your application modular and maintainable.

Understanding Beans in Spring

In Spring, a Bean is simply an object managed by the Spring container. When you mark a class with annotations like @Service, @Repository, or @Controller, Spring creates and manages an instance of that class for you, so you don’t have to do it manually.

Why it matters for FinBank: when our CustomerController needs a CustomerService, we don’t manually write new CustomerService(). Spring injects it via Dependency Injection — making the application more flexible, testable, and decoupled.

Adding Database Support

For this example, we’ll use H2 — an in-memory database — so we can focus on the API without installing external software. We will explore solutions with real databases in the next steps.

We first need to add the H2 dependency in our pom.xml: 

<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-data-jpa</artifactId>
</dependency>
<dependency>
  <groupId>com.h2database</groupId>
  <artifactId>h2</artifactId>
  <scope>runtime</scope>
</dependency>

Configuring H2 Database

Now we need to tell our application that we are using H2 as our database and provide our credentials. Here we are using the default configuration.

Follow the example below in your src/main/resources/application.properties: 

spring.datasource.url=jdbc:h2:mem:finbankdb
spring.datasource.driverClassName=org.h2.Driver
spring.datasource.username=sa
spring.datasource.password=
spring.jpa.database-platform=org.hibernate.dialect.H2Dialect
spring.jpa.hibernate.ddl-auto=update
spring.h2.console.enabled=true

You can access the H2 console at http://localhost:8080/h2-console.

Creating the Customer Entity

To start the implementation, we need to define what a Customer is in our database. For that, we will create the Customer entity.

We will use @Id to identify the ID property of our database, and @GeneratedValue says that its value will be generated automatically. 

package com.finbank.model;

import jakarta.persistence.Entity;
import jakarta.persistence.GeneratedValue;
import jakarta.persistence.GenerationType;
import jakarta.persistence.Id;

@Entity
public class Customer {

  @Id
  @GeneratedValue(strategy = GenerationType.IDENTITY)
  private Long id;
  private String name;
  private String email;

  public Customer() {}

  public Customer(String name, String email) {
    this.name = name;
    this.email = email;
  }

  // getters and setters
}

Creating the Repository

Now that we know what a Customer is (Entity), we can define how to save, update, and query it.

The Repository is a Spring Bean responsible for database operations. Spring Data JPA provides JpaRepository, which gives us CRUD methods out of the box. 

package com.finbank.repository;

import com.finbank.model.Customer;
import org.springframework.data.jpa.repository.JpaRepository;

public interface CustomerRepository extends JpaRepository<Customer, Long> {
}

Creating the Service Layer

The Service layer is where business rules live. This is where we decide how things happen before exposing them to the outside world. In FinBank, this might be where we validate email formats or prevent duplicate accounts. 

package com.finbank.service;

import com.finbank.model.Customer;
import com.finbank.repository.CustomerRepository;
import org.springframework.stereotype.Service;

import java.util.List;

@Service
public class CustomerService {

  private final CustomerRepository repository;

  public CustomerService(CustomerRepository repository) {
    this.repository = repository;
  }

  public List<Customer> findAll() {
    return repository.findAll();
  }

  public Customer findById(Long id) {
    return repository.findById(id)
        .orElseThrow(() -> new RuntimeException("Customer not found"));
  }

  public Customer save(Customer customer) {
    return repository.save(customer);
  }

  public void delete(Long id) {
    repository.deleteById(id);
  }
}

Creating the Controller

The Controller is where HTTP requests are handled. It’s the “face” of your API. Only after defining the data model, database access, and business logic can we safely open it up to API clients. 

package com.finbank.controller;

import com.finbank.model.Customer;
import com.finbank.service.CustomerService;
import org.springframework.web.bind.annotation.*;

import java.util.List;

@RestController
@RequestMapping("/api/customers")
public class CustomerController {

  private final CustomerService service;

  public CustomerController(CustomerService service) {
    this.service = service;
  }

  @GetMapping
  public List<Customer> getAllCustomers() {
    return service.findAll();
  }

  @GetMapping("/{id}")
  public Customer getCustomer(@PathVariable Long id) {
    return service.findById(id);
  }

  @PostMapping
  public Customer createCustomer(@RequestBody Customer customer) {
    return service.save(customer);
  }

  @PutMapping("/{id}")
  public Customer updateCustomer(@PathVariable Long id, @RequestBody Customer customer) {
    Customer existing = service.findById(id);
    existing.setName(customer.getName());
    existing.setEmail(customer.getEmail());
    return service.save(existing);
  }

  @DeleteMapping("/{id}")
  public void deleteCustomer(@PathVariable Long id) {
    service.delete(id);
  }
}

Running the Application

Once you’ve created your Spring Boot project and added the files above, it’s time to run FinBank.

  1. Open the project in your IDE.
  2. Locate the main application class — it’s the one with @SpringBootApplication and the main method.
  3. Click Run in your IDE (often a green triangle button).

You should see something like:

Tomcat started on port(s): 8080 (http) with context path ''
Started FinBankApplication in 3.456 seconds

Spring Boot apps run by default on http://localhost:8080.

Testing the API with Postman

GET /api/customers

{

}

Delete a customer:

DELETE /api/customers/1
