What is Spring Boot?

Spring Boot is a Java framework that simplifies creating production-ready applications by providing:

  • Auto-configuration – detects and configures components automatically
  • Embedded server – no need to manually deploy to Tomcat or another server
  • Starter dependencies – bundles of dependencies for common use cases

For FinBank, this means we can:

  • Quickly set up endpoints for accounts, customers, and transactions
  • Easily connect to databases
  • Focus on business logic instead of boilerplate code

Understanding the Project Structure

A Spring Boot project will have a structure like this: 

src/main/java
 └── com.finbank
     ├── FinBankApplication.java   // Main class
     ├── controller                // REST controllers
     ├── model                     // Entities (Customer, Account, Transaction)
     ├── repository                // Data access layer
     ├── service                   // Business logic

The Main Application Class

The main class contains the @SpringBootApplication annotation. This starts the application with an embedded Tomcat server. 

package com.finbank;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class FinBankApplication {
  public static void main(String[] args) {
    SpringApplication.run(FinBankApplication.class, args);
  }
}

Dependency Injection in Spring Boot

Spring Boot uses Dependency Injection (DI) to manage components. Example – injecting a CustomerService into a CustomerController: 

package com.finbank.controller;

import com.finbank.service.CustomerService;
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/api/customers")
public class CustomerController {

  private final CustomerService customerService;

  public CustomerController(CustomerService customerService) {
    this.customerService = customerService;
  }

  @GetMapping("/{id}")
  public String getCustomer(@PathVariable String id) {
    return customerService.findCustomerById(id);
  }
}

Example – First FinBank Endpoint

Let’s create a basic endpoint to return all accounts.

Model:

package com.finbank.model;

public class Account {
  private String accountNumber;
  private String type;
  private double balance;

  // Constructor, getters, setters
}

Controller:

package com.finbank.controller;

import com.finbank.model.Account;
import org.springframework.web.bind.annotation.*;

import java.util.List;

@RestController
@RequestMapping("/api/accounts")
public class AccountController {

  @GetMapping
  public List<Account> getAllAccounts() {
    return List.of(
        new Account("123", "Checking", 1200.0),
        new Account("456", "Savings", 5000.0)
    );
  }
}

If we start the application and visit: 

http://localhost:8080/api/accounts

We’ll get a JSON list of accounts.

Running the Application

Run: 

mvn spring-boot:run

Or in your IDE, run the main() method in FinBankApplication.
