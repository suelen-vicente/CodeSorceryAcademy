Part 8

Connecting to a Real Database

H2 is great for learning and quick tests, but production needs a persistent database. In this lesson, we connect FinBank to a real database and talk about trade-offs.

← Part 7 Next: Exceptions →

Persistence options (and when)

MySQL / Postgres

Strong relational model, great for typical APIs, transactions, and reporting.

MongoDB

Document model, flexible schema, great for some event-like or document workloads.

Oracle (real-world kingdom)

Powerful optimizer and enterprise features — but demands respect for schema, indexes, and plans.

H2 (learning and tests)

Fast local iteration; not a perfect match for production behavior.

Senior note

“It works on H2” is not proof. Different databases have different SQL dialects, index behavior, and locking semantics.

AI Review Checklist (DB integration)

  • Are connection pool settings sane (timeouts, max pool)?
  • Are transactions defined intentionally?
  • Any N+1 queries or lazy-loading surprises?
  • Does the migration strategy exist (Flyway/Liquibase), or is schema “manual”?

Quest

  2. Define schema migrations (even if minimal).
  3. Load test a single endpoint and inspect query count and latency.
