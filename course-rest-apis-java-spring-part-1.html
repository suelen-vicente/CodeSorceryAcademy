What You’ll Need

Before we dive into coding, let’s make sure you have everything ready. Don’t worry: you don’t need to download random jars manually. We’ll manage dependencies with Maven.

Java Development Kit (JDK)

Install a modern JDK (17+ recommended; 21 is great). Verify with java -version.

Git

Version control tool to track and share your code. Verify with git --version.

Maven

Project management and build automation. Verify with mvn -v.

An IDE

Eclipse (course baseline), IntelliJ IDEA, or VS Code with Java extensions.

Magician tip

Tooling is not “busy work”. It’s the circle of invocation. If your environment is unstable, your learning becomes noise.

Setting Up Your Development Environment

1) Install your IDE

Choose one and stick with it for the course. Consistency beats novelty.

2) Verify your JDK

java -version
javac -version

If your IDE can’t find the JDK, point it to your JDK installation folder.

3) Verify Maven

mvn -v

Maven will handle Spring Boot, JUnit, Lombok, and everything else through dependencies.

4) Lombok (optional, but common)

If you use Lombok, install the Lombok plugin for your IDE and enable annotation processing. We’ll use Lombok carefully (as a tool), not as a way to hide bad design.

Basic Git Workflow

Git will be your best friend when working on projects, both personal and professional. Here’s the basic cycle: 

# Stage changes (prepare them for commit)
git add filename.java
git add .

# Commit changes (save them locally)
git commit -m "Meaningful commit message"

# Push changes (upload to remote repository)
git push origin main

# Pull updates (sync with remote changes)
git pull origin main

Senior habit

Commit small, reviewable diffs. It makes AI-assisted work safer too: you can bisect and rollback with confidence.
