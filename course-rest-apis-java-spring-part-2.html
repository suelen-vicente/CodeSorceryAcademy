Object-Oriented Programming (OOP) for APIs

OOP is the foundation of Java and directly affects how we design our API models, services, and controllers. In a fintech API (like FinBank), design mistakes quickly become security and data integrity problems.

Encapsulation

Encapsulation encourages you to hide an object’s internal state and expose only what’s necessary. This makes classes more secure and easier to understand.

In an API, encapsulation helps prevent exposing sensitive fields (like passwords or internal flags) by mistake.

Example – API User model:

public class User {
  private String username;
  private String email;

  // Constructor
  public User(String username, String email) {
    this.username = username;
    this.email = email;
  }

  // Getter and Setter
  public String getUsername() { return username; }
  public void setUsername(String username) { this.username = username; }

  public String getEmail() { return email; }
  public void setEmail(String email) { this.email = email; }
}

Tip

Don’t create getters and setters without purpose. Some properties don’t need both. Decide what should be accessible and what can be changed to keep your model concise and safe.

Inheritance

Inheritance allows one class to acquire fields and methods from another. It’s useful when paired with polymorphism, but it can also introduce tight coupling if abused.

Example – API Response models:

public class ApiResponse {
  private String status;
  private String message;

  // Constructors, getters, setters

  protected String returnError() {
    return status + " " + message;
  }
}

public class ErrorResponse extends ApiResponse {
  private int errorCode;

  // Constructors, getters, setters
}

public class ErrorTest {
  private ErrorResponse error;
  private ApiResponse response;

  public void test() {
    System.out.println("Error: " + error.returnError());
    System.out.println("Response: " + response.returnError());
  }
}

Here, ErrorResponse inherits from ApiResponse, so both share a common method.

Composition

Composition means building classes using other classes, instead of inheriting from them. In API design, composition is often preferred for flexibility and testability.

Example – Order entity containing a User object:

public class Order {
  private String orderId;
  private User customer; // Composition

  // Constructors, getters, setters
}

Composition also makes automated tests easier: dependencies can be mocked directly, while inheritance often makes behavior harder to isolate.

Polymorphism

Polymorphism allows the same method to take different forms depending on the object. It works closely with inheritance and interfaces.

Example – Payment processing in an API:

public interface PaymentProcessor {
  void processPayment(double amount);
}

public class CreditCardPayment implements PaymentProcessor {
  public void processPayment(double amount) {
    System.out.println("Processing credit card payment: $" + amount);
  }
}

public class PaypalPayment implements PaymentProcessor {
  public void processPayment(double amount) {
    System.out.println("Processing PayPal payment: $" + amount);
  }
}

This helps you avoid multiple if paths: the object itself “knows” how to process a payment. Your API can decide which processor to use at runtime.

Access Modifiers in API Security

You should consider security at the low level of your code. Even access modifiers matter: they are the most basic “security level” of your design. Expose the minimum amount of properties and methods possible — it also makes your code easier for other developers to use correctly.

Modifier Scope API Use Case
public Accessible everywhere API controllers and DTOs
protected Same package and subclasses Base classes (use carefully)
(default) Package-private Internal helpers
private Only within the class Internal state management

== vs equals()

In Java:

  • == compares object references (memory location)
  • equals() compares values

Example – Comparing usernames in an API:

String username1 = new String("admin");
String username2 = new String("admin");

System.out.println(username1 == username2);      // false
System.out.println(username1.equals(username2)); // true

Recommendation: use equals() for value comparisons in API data. Comparing instances is rare and should be intentional.

static and final

static belongs to the class, not the instance — it can be used without an object instance. final cannot be changed after assignment — commonly used for constants.

Example – Defining API constants:

public class ApiConstants {
  public static final String API_VERSION = "v1";
  public static final String BASE_URL = "/api/" + API_VERSION;
}
