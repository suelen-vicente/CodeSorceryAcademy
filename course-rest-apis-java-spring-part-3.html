Collections & Data Processing for APIs

In this lesson, we’ll explore Java’s Collections Framework and how it’s essential for handling data in backend services. In a REST API, collections are everywhere — from processing incoming JSON arrays to building paginated lists for clients.

Introduction to the Collections Framework

The Collections Framework is a set of classes and interfaces that manage groups of objects. The most common ones for API development are:

  • List – Ordered collection, allows duplicates.
  • Set – Unordered collection, no duplicates.
  • Map – Key-value pairs, no duplicate keys.

Magician hint

Collections are “invisible infrastructure”. If you pick the wrong one, your API might still work — but it will cost performance, clarity, or correctness.

Lists – Ordered Data

Lists are perfect when the order matters and duplicates are allowed. In a real-world API, we might use a List to store a customer’s transaction history. 

import java.util.ArrayList;
import java.util.List;

public class TransactionService {
  public List<String> getRecentTransactions() {
    List<String> transactions = new ArrayList<>();
    transactions.add("Deposit $500");
    transactions.add("Withdrawal $100");
    transactions.add("Deposit $500"); // duplicates allowed
    return transactions;
  }
}

If the API endpoint /customers/{id}/transactions returns this list, the order will match the order in which transactions occurred.

Sets – Unique Data

Sets are great when duplicates are not allowed. In our example, we could use a Set to store unique customer IDs who have subscribed to premium services. 

import java.util.HashSet;
import java.util.Set;

public class CustomerService {
  private Set<String> premiumCustomers = new HashSet<>();

  public void addPremiumCustomer(String customerId) {
    premiumCustomers.add(customerId);
  }

  public Set<String> getPremiumCustomers() {
    return premiumCustomers;
  }
}

A Set ensures each customer ID appears only once, even if the API tries to add them multiple times.

Maps – Key-Value Storage

Maps are ideal for fast lookups. For example, we could map an account number to its balance. 

import java.util.HashMap;
import java.util.Map;

public class AccountService {
  private Map<String, Double> accountBalances = new HashMap<>();

  public void updateBalance(String accountNumber, double balance) {
    accountBalances.put(accountNumber, balance);
  }

  public Double getBalance(String accountNumber) {
    return accountBalances.get(accountNumber);
  }
}

In a REST API, this can back endpoints like /accounts/{number}/balance.

hashCode() and equals() in Collections

For Sets and Maps, hashCode() and equals() determine whether two objects are considered the same. These methods should be implemented by you, so your code defines the equality criteria and the hash code.

Example: customers are unique by their ID, not by their name. So the hash code uses the ID and equals compares IDs. 

public class Customer {
  private String id;
  private String name;

  // constructor, getters, setters

  @Override
  public int hashCode() {
    return id.hashCode();
  }

  @Override
  public boolean equals(Object obj) {
    if (this == obj) return true;
    if (!(obj instanceof Customer)) return false;
    Customer other = (Customer) obj;
    return this.id.equals(other.id);
  }
}

This prevents duplicate customers in a Set<Customer> even if they have different names.

Iterating Through Collections

Sometimes, you’ll need to loop through elements to transform or filter them before returning them in the API. 

import java.util.List;

public class ReportService {
  public void printTransactions(List<String> transactions) {
    for (String tx : transactions) {
      System.out.println(tx);
    }
  }
}

In REST APIs, you might filter transactions to only show those above a certain amount.

Streams API – Modern Data Processing

Java Streams make it easier to process collections declaratively. It allows you to construct a chain of operations that can be applied to a collection.

Example: Find all high-value transactions over $1,000.

import java.util.List;
import java.util.stream.Collectors;

public class TransactionFilter {
  public List<Double> filterHighValue(List<Double> transactions) {
    return transactions.stream()
        .filter(amount -> amount > 1000)
        .collect(Collectors.toList());
  }
}

In an API, this could power an endpoint like /transactions/high-value.
