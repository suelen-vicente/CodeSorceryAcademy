Part 10

Input Validation

Now we’ll make sure bad data never even makes it to our service layer by adding input validation. Validation is not a “nice-to-have”. It’s part of the API contract.

← Part 9 Next: DTOs →

Edge validation (Bean Validation)

Use validation annotations on request DTOs. Keep domain validation rules explicit too (some rules are too complex for annotations). 

import jakarta.validation.constraints.Email;
import jakarta.validation.constraints.NotBlank;

record CreateCustomerRequest(
  @NotBlank String name,
  @Email String email
) {}

Then ensure your controller validates: @Valid + centralized error mapping.

AI Review Checklist (Validation)

  • Are validation rules on request DTOs (not entities)?
  • Are messages safe and consistent?
  • Are cross-field validations handled (e.g., date ranges)?
  • Does validation happen before calling the service?
